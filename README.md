# MusicShop

Учебный интернет-магазин виниловых пластинок: каталог с поиском и фильтрами, корзина, оформление заказа, регистрация и вход. Для администраторов есть форма добавления пластинки в каталог.

Стек по сути один — сервер на **Python (FastAPI)** отдаёт HTML через **Jinja2**, стили на **Tailwind** (подключение с CDN, без отдельной сборки фронта). Данные лежат в **PostgreSQL**, доступ через **psycopg2**. Сессии пользователя — в cookie (Starlette SessionMiddleware).

## Структура репозитория

| Папка | Содержимое |
|--------|------------|
| `backend/` | Код приложения: `app/main.py` (маршруты), `app/db.py` (запросы к БД), шаблоны в `app/templates/`, тесты в `tests/` |
| `database/` | SQL-схема, миграции, скрипты для pgAdmin / psql |
| `docs/` | Вспомогательные материалы, например регрессионный чек-лист в CSV |
| `Musicstore/` | У многих в команде здесь лежит `.env` с `DATABASE_URL` — путь к файлу смотри в `backend/README` |

Истории коммитов и задачи ведём в **GitHub** (Issues подойдут и для багов, и для тест-кейсов).

## Что нужно на машине

- **Python** 3.10+ (у нас в проекте ориентир на 3.12)
- **PostgreSQL** с созданной базой, например `MusicShopDB`, и применённой схемой из `database/`
- По желанию — **pgAdmin** или другой клиент, если удобнее кликать, чем `psql`

Секреты в репозиторий не кладём: строка подключения к БД и ключ сессии задаются через переменные окружения (файл `.env` в корне или в `Musicstore/` — см. `backend/app/main.py`, там перечислены кандидаты путей).

## Быстрый старт

1. Клонировать репозиторий, зайти в каталог проекта.
2. Создать виртуальное окружение и поставить зависимости:
   ```bash
   python3 -m venv .venv
   source .venv/bin/activate   # Windows: .venv\Scripts\activate
   pip install -r backend/requirements.txt
   ```
3. Настроить `.env`: как минимум `DATABASE_URL` в формате `postgresql://user:password@localhost:5432/MusicShopDB` (или с `postgresql+psycopg2://` — в коде это нормализуется). Для локальной разработки можно оставить `SESSION_SECRET` дефолтным из кода, но для чего-то публичного лучше свой случайный ключ.
4. Поднять схему БД — см. **`database/README.md`**. Для демо можно прогнать `database/deploy/musicshop_seed.sql` после создания таблиц (осторожно: скрипт чистит часть данных каталога перед вставкой примеров).
5. Запустить сервер из **корня** репозитория:
   ```bash
   uvicorn backend.app.main:app --reload --port 8000 --reload-dir backend
   ```
   Последний флаг уменьшает лишние перезапуски из-за изменений в `.venv`.

Открыть в браузере: `http://127.0.0.1:8000` — редирект на поиск. Документация API в стиле Swagger: `http://127.0.0.1:8000/docs` (для отчётов и ручных проверок удобно).

## Тесты

Юнит-тесты на утилиты и интеграционные на регистрацию/вход лежат в `backend/tests/`. Подробности и команды — в **`backend/README.md`**. Интеграционные тесты auth ожидают живую БД с `DATABASE_URL`.

Корневой `pytest.ini` задаёт `pythonpath`, чтобы `pytest` можно было гонять из корня проекта.

## Чек-лист регрессии

В `docs/regression_checklist_musicshop.csv` — таблица проверок для ручного прогона перед сдачей или после правок в каталоге и заказах. Импорт в Google Таблицы или Excel, кодировка UTF-8.

## Лицензия и дисклеймер

Проект учебный; код и схема БД могут меняться под задания курса. Для production нужны отдельный аудит безопасности, настройка HTTPS, секретов и резервного копирования БД.
